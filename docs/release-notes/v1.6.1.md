# Toolport v1.6.1

**Windows auto-update fix when MCP clients hold the gateway open.**

If you use Cursor, Codex, or another client that spawns `toolport-gateway.exe`, the
in-app updater could fail because NSIS could not overwrite the locked install-dir binary.
This patch fixes that without asking you to close your editor.

---

## What changed

- **Versioned client gateway** — on launch, Toolport publishes
  `toolport-gateway-{version}.exe` under `%APPDATA%\Roaming\Conduit\bin` and re-points
  MCP client configs there instead of the install-dir copy.
- **Pre-update gateway stop** — before downloading/installing an update, Toolport stops
  only spawned `toolport-gateway` / `conduit-gateway` processes so the installer can
  replace files. Your MCP clients stay open and reconnect after restart.

---

## Upgrade

**Desktop (Windows):** Settings → About → Check for updates, or download from
[Releases](https://github.com/tsouth89/toolport/releases).

**Docker / headless:** No change required for this patch unless you rebuild from source;
`ghcr.io/tsouth89/toolport-gateway:latest` on `main` already includes the gateway binary
from the merged fix.
