# Toolport v1.6.2

**Fixes Windows install when MCP clients hold `toolport-gateway.exe` open.**

v1.6.1 added versioned gateway paths and pre-update kills for the **in-app updater**,
but only after you were already on 1.6.1. Manual installer downloads and the
**1.6.0 → 1.6.1** upgrade still failed with “Error opening file for writing” when
Cursor or another client had spawned the gateway.

This patch adds an **NSIS pre-install hook** that terminates `toolport-gateway.exe`
and `conduit-gateway.exe` before the installer copies files. Your editors can stay open;
only the gateway child exits.

---

## Upgrade

**From 1.6.0 or 1.6.1:** download the Windows installer from
[Releases](https://github.com/tsouth89/toolport/releases) and run it — no need to close
Cursor first.

**Already on 1.6.2+:** Settings → About → Check for updates.
